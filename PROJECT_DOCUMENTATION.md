# XmartyCreator 2.0 - Project Documentation

## Project Overview
XmartyCreator 2.0 is a modern, dynamic, AI-powered EdTech platform designed for creators and software engineers. It bridges the gap between learning and architectural execution through hands-on system architecture challenges and live mentoring.

The platform relies on a dual-domain architecture:
1. **Main Domain (`src/app`)**: The public-facing site where users browse courses, community features, read about the platform, and take mega-quizzes.
2. **Support Domain (`supportdomain`)**: A custom-built admin Content Management System (CMS) that allows non-technical team members to dynamically update the text, images, lists, and SEO metadata across the entire Main Domain without requiring codebase deployments.

## Known Architecture & Flow
- **Data Flow**: Admin updates values in SupportDomain -> Values are saved to MongoDB as `content_blocks` -> Main site `useContentBlock` fetches values via `/api/mongodb-gateway`.
- **Media**: Image fields can handle direct URLs or JSON strings. Handled via the `getImageUrl` parser in the frontend code.
- **Lists**: `list` types in the schema store JSON arrays of objects. `parseList()` is used to safely map these array elements (e.g., Features in the Community section, Stats in the Hero section).
